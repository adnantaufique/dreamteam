# Dreamteam — Learnings Store

Curated, human-reviewed defaults the **Caster** consults before a run (`references/caster.md`). Entries are **evidence-tagged advisory defaults, overridable** — never hard rules. Persisted by `references/retro.md`; promoted to skill edits only via the human gate. Seeded empty.

| date | project_key | profile/task_kind | learning (evidence) | applied? |
|---|---|---|---|---|
| | | | | |

**Schema → column mapping** (how a `retro.md` record becomes a row): `project_key ← record.project_key` · `profile/task_kind ← record.profile + task_kind` · `learning (evidence) ← each change.delta + its change.evidence` (plus notable `worked[]` items) · `applied? ← "no"` until the human promotes it (the human gate). One row per `change[]` entry; the now-numeric `confidence` (0.3–0.9) still travels in the learning cell.

**Project vs global scope** (the `project_key` column): a **git-remote hash** = a **PROJECT** learning — consulted **only** when the current repo's git-remote hash matches, so a learning from project A no longer leaks into project B. A **`global`** value (or a blank/legacy cell) = a **GLOBAL** learning — consulted on **every** run. The Caster therefore consults **global + current-project** entries only (`references/caster.md` step 0). **Optional promotion (human-gated):** a PROJECT learning that recurs across **2+ distinct projects** with **average `confidence` ≥ 0.8** MAY be promoted to global (set its `project_key → global`) — still an advisory default, never a hard rule, in the same single-markdown store.

**Agent scouting ledger** — a second table in the same store: **per-agent casting evidence**. The Caster picks agents by *description*, and with dozens to hundreds of agents potentially installed a description doesn't predict **gate outcomes**; the **run record** (`loop.md` §Report buffer, including the dispatch-reliability tally) captures that truth for one run, and this ledger **accumulates it per agent**, so casting improves with use. The Caster consults it alongside the learnings above (`references/caster.md` step 0); rows are **advisory defaults, never bans** — a poor record surfaces as rationale for preferring a fitting alternative, never excludes an agent outright. Seeded empty.

**Written only by the retro** (`references/retro.md`), from the run record (`loop.md` §Report buffer, including the dispatch-reliability tally) — **no row without run evidence** (the store's honesty rule, unchanged). An update **increments the counts of an existing matching row** (same `agent` + `project_key` + `profile`); no match → append one. **The `profile/task_kind` cell matches on `profile` ALONE** — its task_kind tail is a **descriptive annotation** the retro may update (e.g. the latest kinds seen), **never a match criterion**, so rows never fragment per task_kind. **Record → column mapping:** `dispatches` ← the agent's dispatches that run · `first-try passes` ← workstreams cleared with no fix iteration (the run record's **first-try misses**, inverted) · `fix re-dispatches` ← the run record's **fix-iteration counts** · `recovery re-dispatches (drops)` ← the reliability tally's drops by failure class (**dropped-without-returning** / **returned-but-inadmissible**) + their **recovery re-dispatches** · `refuted predictions` ← its **refuted-and-dropped** refuter outcomes, already keyed to the **emitting reviewer** (`loop.md` §Report — the names match verbatim); anything else worth carrying (e.g. a schema-cap **schema-free re-run**) goes in `notes`. Persisting here is what keeps the run itself stateless: the run record stays report output only — **this file is the already-sanctioned persistence point**.

**Ledger scope + confidence:** `project_key` reuses the PROJECT-vs-`global` semantics above **verbatim** — the Caster consults **global + current-project rows only**. **Promotion analogue (human-gated as ever):** rows for the same agent + `profile` recurring across **2+ distinct projects**, with **every participating row at ≥3 dispatches** (the sample-size analogue of the learnings **`confidence` ≥ 0.8** bar), MAY be promoted to global — the participating rows **MERGE into one `global` row with SUMMED counts**. **Sample size IS the confidence signal — no separate confidence column:** `dispatches` says how much the counts are worth, and the Caster **ignores a row with fewer than 3 dispatches as noise** (`references/caster.md` step 0).

Per-environment: this file lives in the installed skill; learnings are local unless the human promotes them upstream. (On Codex the synced copy lives alongside the `AGENTS.md` pointer; on Gemini under `~/.gemini/agents/dreamteam/` — see `references/platforms.md`.)
