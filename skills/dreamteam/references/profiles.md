# Dreamteam ‚Äî Domain Profiles (fast-path dreamteam rosters)

If the task clearly matches a row, use that dreamteam ‚Äî no Caster-agent hop. `generic` is the table's fallback; anything richer or cross-domain ‚Üí the Caster agent (`caster.md`).

**Reality Checker is in every gate** ‚Äî it verifies claims against evidence (tests for code; data‚Üîclaims for research; explanation‚Üîsource for tutor; a reproduced failure for debug; re-derived a11y for ux-designer).

Fast-path rows list planner / producers / gate / workstreams only ‚Äî they **inherit `gate_policy` defaults** (`min_pass` = all reviewers, `max_fix_iterations` = 2) and the full dreamteam-manifest field names from `caster.md`.

Fast-path rows carry default tiers (mechanical‚Üícheap, producers‚Üístandard, reviewers‚Üícapable); max is never a default. Tiers are abstract (resolved per `references/platforms.md`; F1 Claude names accepted via back-compat), overridable via `--models`/`--cost` and adjustable by the Caster agent.

Code-producing producers (mobile-dev build ¬∑ web backend/frontend ¬∑ devops ¬∑ qa ¬∑ generic ¬∑ ml-dev build ¬∑ debug fix ¬∑ ux-designer implement) compose **`ponytail`** (minimal-code / anti-over-engineering) **and `karpathy-guidelines`** (surface assumptions ¬∑ surgical/minimal-diff changes ¬∑ verifiable success criteria) when installed ‚Äî attached to their `skills[]` on **both the fast-path and the agent path** (each `find-skills`-gated independently; the producer works without any that's absent). The gate flags over-engineering **regardless** ‚Äî that's dreamteam's native minimal-code principle (see `SKILL.md`); `ponytail` + `karpathy-guidelines` just enforce it harder when installed. The ai-research *writing* roles and the **tutor** roles produce papers/experiments/explanations, not production code (as does ux-designer's *design* sub-role), so these enforcers apply only to a genuinely code-producing sub-role.

**mobile-dev** drops the design-architect by default ‚Äî `Mobile App Builder` already covers iOS + Android + cross-platform, so the design roster is just `UI Designer`. For visually/architecturally complex mobile work the Caster can re-add a design-architect (the "rosters are defaults; the Caster can adjust" escape hatch below).

**audit** producers depend on mode (set by `references/audit.md` ‚Äî read-only by default): in *bugs* mode the dimension reviewers run **as producers**; in *map* mode the explorers/architects + a synthesizer run. It overrides two manifest fields beyond the defaults ‚Äî `gate_policy.devils_advocate_on_unanimous: true` and `audit_policy.max_parallel_reviewers` (per-`--depth` defaults: shallow 2 / module 4 / exhaustive 8, ceiling 12). See `references/audit.md` for modes, `--depth`, and the read-only contract.

**ml-dev** is ML *development* (training / eval / data pipelines, model code) ‚Äî distinct from **ai-research**'s research-salvage / paper / hypothesis work. The `mle-workflow` skill (bundled) + `build-error-resolver` / `pytorch-build-resolver` (bundled) are present by default; `pytorch-build-resolver` joins the build-fix loop only for training / CUDA errors. `python-expert` is used when bundled, else recommend-only ‚Äî the gate (methodology-reviewer + Reality Checker, + Security Engineer when infra/data-sensitive) is unchanged either way.

**debug** skips plan-writing and goes **reproduce-first** ‚Äî the *same* SKILL.md spine step-2 carve-out as `audit` (skip wrapper + plan-writing); the failure *is* the spec, so there is no plan to brainstorm or write. It composes `systematic-debugging` when installed (`find-skills`-gated); absent ‚Üí `root-cause-analyst` investigates natively and the Caster records a `recommendations[]` entry ‚Äî never a block. **Unlike audit, debug integrates** (it lands the fix). Its gate is non-standard: the Reality Checker must show the failing case **RED before / GREEN after** with a regression test (reproduce-then-resolve, `gate.md` ¬ß3) ‚Äî a fix whose test passes even against the *original* unfixed code is rejected as vacuous.

**ux-designer** is design-led: `UI Designer` composes `ui-ux-pro-max` (a **depended** skill) when installed; absent ‚Üí UI Designer designs natively and the Caster records a `recommendations[]` entry, and **a11y stays NON-WAIVABLE regardless**. The **redesign** sub-role adds `deep-research-agent` to read the existing UI/codebase (NO Understand-Anything); **greenfield omits it** (N/A, not degradation). The gate branches **spec-only vs code-emitted** ‚Äî the Code Reviewer + the code enforcers (`ponytail` / `karpathy-guidelines`) attach only when implementation code is produced.

**tutor** ("understand anything, then explain it simply") is self-contained ‚Äî **no external skill** (`deep-research-agent` understands ‚Üí `Technical Writer` explains). If `deep-research-agent` is recommend-only rather than bundled, fall back to the host + `recommend.md` and flag. Its gate pairs explanation‚Üîsource accuracy with a clarity/simplicity rubric, and **the source wins on any conflict** (no lies-to-children ‚Äî never simplify into something false).

## Matching
Map the task to a profile by intent/keywords: mobile / Android / iOS / React Native / Flutter / Expo / cross-platform ‚Üí **mobile-dev**; website / API / full-stack ‚Üí **web**; experiment / paper / model / dataset / hypothesis ‚Üí **ai-research**; train / fine-tune / ML pipeline / dataset loader / eval harness / pytorch / model code ‚Üí **ml-dev** (ML *development*, distinct from ai-research's research-salvage); CI / deploy / infra / k8s / terraform ‚Üí **devops**; test-coverage / regression / QA sweep ‚Üí **qa**; debug / fix this bug / why is this failing / stack trace / reproduce a regression ‚Üí **debug**; design system / wireframe / redesign / UX audit / a11y of a UI / component library ‚Üí **ux-designer** ("build the app/site" stays ‚Üí **web** / **mobile-dev**, with design as a sub-role); explain X / teach me / ELI5 / how does X work ‚Üí **tutor** ("map the codebase / understand this repo" stays ‚Üí **audit**); audit / review / find bugs / map the codebase / understand this repo ‚Üí **audit** (test-coverage / regression stays ‚Üí **qa**). No clear match ‚Üí the Caster agent (do not force a profile).

`--profile android` is accepted as a back-compat alias for **mobile-dev**.

The rosters are defaults ‚Äî a `--roster` override or the Caster agent can adjust them (e.g. add Security Engineer, swap a builder).
